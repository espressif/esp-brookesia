## 线程分析器

打印线程分析器信息：

```bash
debug_thread
```

使用自定义排序和采样持续时间：

```bash
# 设置主排序方式（none|core，默认：core）
debug_thread -p core

# 设置次排序方式（cpu|priority|stack|name，默认：cpu）
debug_thread -s cpu

# 设置采样持续时间（毫秒，默认：1000）
debug_thread -d 2000

# 组合使用
debug_thread -p core -s cpu -d 2000
```

参数说明：

| 参数 |         说明         |               可选值               | 默认值 |
| ---- | -------------------- | ---------------------------------- | ------ |
| `-p` | 主排序方式           | `none`, `core`                     | `core` |
| `-s` | 次排序方式           | `cpu`, `priority`, `stack`, `name` | `cpu`  |
| `-d` | 采样持续时间（毫秒） | 任意正整数                         | `1000` |

## 时间分析器

打印时间分析器报告：

```bash
debug_time_report
```

清除所有时间分析器数据：

```bash
debug_time_clear
```

输出示例：

```
=== Performance Tree Report ===
(Unit: ms)
Name                             |  calls |      total |       self |        avg |        min |        max | %parent |  %total
------------------------------------------------------------------------------------------------------------------------------
|- app_main                      |      1 |     922.31 |     922.31 |     922.31 |     922.31 |     922.31 |   92.67% |   92.67%
`- console_start                 |      1 |      72.93 |      72.93 |      72.93 |      72.93 |      72.93 |    7.33% |    7.33%
===============================
```

> [!NOTE]
> 时间分析器数据需要在代码中显式调用接口来采集：
> - `BROOKESIA_TIME_PROFILER_SCOPE(name)`：作用域计时
> - `BROOKESIA_TIME_PROFILER_START_EVENT(name)`：开始计时
> - `BROOKESIA_TIME_PROFILER_END_EVENT(name)`：结束计时
